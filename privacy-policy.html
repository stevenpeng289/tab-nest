TabNest Privacy Policy

TabNest is a Chrome extension that replaces the browser's new tab page with a workspace for organizing tabs, bookmark folders, quick links, and saved sessions.

1. Information TabNest handles

To provide its features, TabNest may access or locally store the following information:

  • Open tab metadata, such as tab titles, URLs, favicon URLs, active state, tab index, and window ID
  • Saved sessions created by the user, including tab titles and URLs stored in chrome.storage.local
  • Bookmark metadata from folders the user chooses to display, including bookmark folder titles, bookmark titles, URLs, and folder hierarchy
  • Quick links created by the user
  • Saved-for-later items created by the user
  • User preferences such as language, theme, background settings, view mode, and bookmark board layout preferences
  • Custom background images selected by the user, stored locally in the browser

2. How the information is used

  • To display and organize the user's currently open tabs and windows
  • To show selected bookmark folders as bookmark boards and let the user navigate subfolders or remove bookmarks
  • To let the user close, move, reorder, stash, and restore tabs and sessions
  • To save preferences and personalize the new tab page
  • To show quick links, duplicate detection, and saved tab content chosen by the user

3. Where data is stored

TabNest stores its settings and saved content locally in the browser using chrome.storage.local. TabNest does not require an account and does not send this locally stored data to a developer-operated server.

4. External requests and third parties

TabNest does not operate its own backend service and does not send locally stored sessions, bookmark board configuration, preferences, or quick links to any developer-managed server.

The extension uses local browser storage for saved data and may use Chrome's built-in favicon retrieval mechanism to display website icons. Any network activity related to open tabs is initiated by the websites the user chooses to visit in Chrome, not by a TabNest backend.

5. Data sharing

TabNest does not sell user data. TabNest does not share locally stored session data, bookmark board configuration, quick links, or preferences with the developer or with unrelated third parties.

6. Data retention and deletion

  • Saved sessions, bookmark board configuration, quick links, saved-for-later items, and preferences remain in local browser storage until the user edits, deletes, or clears them
  • Removing the extension will generally remove the extension's locally stored data from Chrome

7. Security

TabNest is designed to operate locally in the browser and does not maintain a remote database of user content.

8. Children's privacy

TabNest is not directed to children and is intended as a general productivity tool for browser users.

9. Changes to this policy

This policy may be updated from time to time. Any material changes should be reflected on this page by updating the "Last updated" date.

10. Contact

For questions about this policy, you can contact the developer through the project repository: GitHub Issues.

中文

TabNest 隐私政策

TabNest 是一个 Chrome 扩展，用于替换浏览器新标签页，并提供标签整理、书签工作台、快捷入口和会话收纳等功能。

1. TabNest 处理的信息

为了提供核心功能，TabNest 可能会访问或在本地存储以下信息：

  • 当前打开标签页的元数据，例如标签标题、URL、favicon URL、激活状态、标签顺序和窗口 ID
  • 用户主动创建的会话收纳数据，包括保存在 chrome.storage.local 中的标签标题和 URL
  • 用户主动选择展示的书签目录及其子项信息，包括目录名称、书签标题、URL 和层级关系
  • 用户主动创建的快捷入口
  • 用户主动保存的“稍后处理”标签
  • 语言、主题、背景、视图模式、书签工作台布局等偏好设置
  • 用户选择上传的自定义背景图片，该图片保存在浏览器本地

2. 信息的用途

  • 用于展示并整理用户当前打开的标签页和窗口
  • 用于把用户选择的书签目录展示为书签工作台，并支持进入子目录或删除单个书签
  • 用于实现关闭、移动、拖拽排序、收纳和恢复标签/会话等功能
  • 用于保存界面偏好和个性化设置
  • 用于展示用户选择保存的快捷入口、重复标签识别结果和稍后处理内容

3. 数据存储位置

TabNest 的设置和保存内容默认存储在浏览器本地的 chrome.storage.local 中。TabNest 不要求用户注册账号，也不会把这些本地存储的数据发送到开发者自建服务器。

4. 外部请求与第三方服务

TabNest 不运行自有后端服务，也不会把本地存储的会话、书签工作台配置、偏好设置或快捷入口发送到开发者自建服务器。

当前版本的扩展使用浏览器本地存储保存数据，并可能通过 Chrome 内置的 favicon 获取机制来显示网站图标。与打开中的网页相关的网络请求，来自用户在 Chrome 中主动访问的网站，而不是 TabNest 自身的后端服务。

5. 数据共享

TabNest 不出售用户数据。TabNest 不会把本地保存的会话、书签工作台配置、快捷入口、偏好设置等内容共享给开发者或无关第三方。

6. 数据保留与删除

  • 会话、书签工作台配置、快捷入口、稍后处理内容和偏好设置会保留在本地浏览器存储中，直到用户主动修改、删除或清理它们
  • 卸载扩展后，Chrome 通常会删除该扩展对应的本地存储数据

7. 安全性

TabNest 设计为主要在浏览器本地运行，不维护远程用户内容数据库。

8. 儿童隐私

TabNest 不是面向儿童设计的产品，而是面向普通浏览器用户的效率工具。

9. 政策更新

本政策可能会不时更新。如有重要变更，应通过更新本页面中的“最后更新日期”进行说明。

10. 联系方式

如对本政策有疑问，可通过项目仓库联系开发者： GitHub Issues。
